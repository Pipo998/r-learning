Creating many strings from data

str_c()

Takes any number of vectors as arguments and returns a character vector:

str_c("x", "y")
[1] "xy"
str_c("x", "y", "z", sep = " ")
[1] "x y z"
str_c("Hello ", c("John", "Susan"))
[1] "Hello John"  "Hello Susan"

Documentation: str_c() similar to paste0() - BUT designed to be used with mutate():

df <- tibble(name = c("Flora", "David", "Terra", NA))

df |> mutate(greeting = str_c("Hi ", name, "!"))
# A tibble: 4 × 2
  name  greeting 
  <chr> <chr>    
1 Flora Hi Flora!
2 David Hi David!
3 Terra Hi Terra!
4 <NA>  <NA>

Missing values can be displayed in another way using coalesce() to replace them:

# Documentation 'Value' - Vector with same type and size
df |>
  mutate(
    greeting1 = str_c("Hi ", coalesce(name, "you"), "!"),
    greeting2 = coalesce(str_c("Hi ", name, "!"), "Hi!")
  )
# A tibble: 4 × 3
  name  greeting1 greeting2
  <chr> <chr>     <chr>    
1 Flora Hi Flora! Hi Flora!
2 David Hi David! Hi David!
3 Terra Hi Terra! Hi Terra!
4 <NA>  Hi you!   Hi!

str_glue()

Mixing many fixed and variable strings with str_c() requires typing a lot of "s. Alternative: Use glue package via str_glue() - special feature: Anything inside {} will be evaluated like it’s outside of quotes (see ?str_glue):

df |>
  mutate(greeting = str_glue("Hi {name}"))
# A tibble: 4 × 2
  name  greeting
  <chr> <glue>  
1 Flora Hi Flora
2 David Hi David
3 Terra Hi Terra
4 <NA>  Hi NA

str_flatten()

str_flatten() works with summarize() - takes character vector and combines each element of the vector into single string:

str_flatten(c("x", "y", "z"))
[1] "xyz"
str_flatten(c("x", "y", "z"), collapse = ", ")
[1] "x, y, z"
str_flatten(c("x", "y", "z"), collapse = ", ", last = ", and ")
[1] "x, y, and z"
# Works well with summarize:

df <- tribble(
  ~ name, ~ fruit,
  "Carmen", "banana",
  "Carmen", "apple",
  "Marvin", "nectarine",
  "Terence", "cantaloupe",
  "Terence", "papaya",
  "Terence", "mandarin"
)

df |> 
  group_by(name) |>
  summarize(fruits = str_flatten(fruit, collapse = ", "))
# A tibble: 3 × 2
  name    fruits                      
  <chr>   <chr>                       
1 Carmen  banana, apple               
2 Marvin  nectarine                   
3 Terence cantaloupe, papaya, mandarin

Extracting data from strings

Common that multiple variables getting crammed together into a single string.

Four different tidyr functions for extraction: + df |> separate_longer_delim(col, delim) + df |> separate_longer_position(col, width) + df |> separate_wider_delim(col, delim, names) + `df |> separate_wider_position(col, widths)```

Similar with pivot_longer() and pivot_wider() - make input data frame longer creating new rows and wider by generating new columns.

delim splits up string with delimiter like ", " or " ".; position splits at specific widths, like c(3, 5, 2).

Separating into rows

Useful when number of components varies from row to row. Most common case is splitting based on delimiter:

df1 <- tibble(x = c("a,b,c", "d,e", "f"))

df1 |>
  separate_longer_delim(x, delim = ",")
# A tibble: 6 × 1
  x    
  <chr>
1 a    
2 b    
3 c    
4 d    
5 e    
6 f    
df2 <- tibble(x = c("1211", "131", "21"))

df2 |>
  separate_longer_position(x, width = 1)
# A tibble: 9 × 1
  x    
  <chr>
1 1    
2 2    
3 1    
4 1    
5 1    
6 3    
7 1    
8 2

Separating into columns

Useful when fixed numbers of components in each string, and want to spread into columns:

df3 <- tibble(x = c("a10.1.2022", "b10.2.2011", "e15.1.2015"))

df3 |>
  separate_wider_delim(x,
                       delim = ".",
                       names = c("code", "edition", "year")
                       )
# A tibble: 3 × 3
  code  edition year 
  <chr> <chr>   <chr>
1 a10   1       2022 
2 b10   2       2011 
3 e15   1       2015 
# Can also omit specific piece by NA

df3 |>
  separate_wider_delim(x,
                       delim = ".",
                       names = c("code", NA, "year")
                       )
# A tibble: 3 × 2
  code  year 
  <chr> <chr>
1 a10   2022 
2 b10   2011 
3 e15   2015 
# separate_wider_position() needs specifying width of each column

df4 <- tibble(x = c("202215TX", "202122LA", "202325CA"))

df4 |>
  separate_wider_position(
    x,
    widths = c(year = 4, age = 2, state = 2)
  )
# A tibble: 3 × 3
  year  age   state
  <chr> <chr> <chr>
1 2022  15    TX   
2 2021  22    LA   
3 2023  25    CA

Subsetting

Extract parts of string via str_sub(string, start, end). + start and end are position where substring starts and ends (inclusive arguments). Length of returned string: end - start + 1:

x <- c("Apple", "Banana", "Pear")

str_sub(x, start = 1, end = 3)
[1] "App" "Ban" "Pea"
# Can use negative values to count back from the end of the string:
# -1 is last character, -2 is second to last character

str_sub(x, start = -3, end = -1)
[1] "ple" "ana" "ear"
# Too short string; str_sub returns as much as possible

str_sub("a", start = -3, end = -1)
[1] "a"
babynames |>
  mutate(
    first = str_sub(name, start = 1, end = 1),

Non-English text

Encoding

Underlying representation of strings in R accessed via charToRaw():

charToRaw("Hadley")
[1] 48 61 64 6c 65 79
# Each represents one letter
# mapping from hexadecimal numbers to character is called encoding
# Encoding is ASCII

For more read http://kunststube.net/encoding/.

Letter variations

u <- c("\u00fc", "u\u0308")
