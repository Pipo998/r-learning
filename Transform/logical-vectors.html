Logical Vectors

Introduction

Logical vectors can only be: TRUE, FALSE, NA.

Simple way of creating logical vectors \(\rightarrow\) numeric comparisons

Two useful functions \(\rightarrow\) ìf_else() and case_when().

Boolean algebra

Can combine multiple vectors using Boolean algebra:

&“and”,|“or”,!“not”,xor()“exclusive or”.

Note

xor() is true if x is true, or if y is true, but not both.

Missing values

df <- tibble(x = c(TRUE, FALSE, NA))

df |>
  mutate(
    and = x & NA,
    or = x | NA
  )
# A tibble: 3 × 3
  x     and   or   
  <lgl> <lgl> <lgl>
1 TRUE  NA    TRUE 
2 FALSE FALSE NA   
3 NA    NA    NA

Missing value in logical vector can be TRUE or FALSE - we don’t know. + NA | TRUE: Must be TRUE(either or is true) + NA | FALSE: Is NA - don’t know if NA is TRUE or FALSE. + NA | NA: is NA

  • NA & TRUE: is NA- don’t know if NA is TRUE or FALSE
  • NA & FALSE: is FALSE - at least one condition is FALSE
  • NA & NA: is NA

Summaries

Logical summaries

Two main logical summaries: any() and all().

any(x) is equivalent to | \(\rightarrow\) returns TRUE if there any TRUE’s in x.

all(x) is equivalent to & \(\rightarrow\) returns TRUE if all values of x are TRUE.

Most summary function include na.rm = TRUE.

flights |>
  group_by(year, month, day) |>
  summarize(
    all_delayed = all(dep_delay <= 60, na.rm = TRUE), 
    any_long_delay = any(arr_delay >= 300, na.rm = TRUE),
    .groups = "drop"
  )

Conditional transformation

Two important tools: if_else() and case_when().

if_else()

Note

Main advantage to base R’s ìfelse() is that with dyplr::if_else() can choose what happens to missing values, and it’s more likely to give meaningful error when variable types not compatible.

Format: if_else(condition, true, false, missing)

x <- c(-3:3, NA)
if_else(x > 0, "+ve", "-ve")
[1] "-ve" "-ve" "-ve" "-ve" "+ve" "+ve" "+ve" NA   
# How to deal with NA
if_else(x > 0, "+ve", "-ve", "???")
[1] "-ve" "-ve" "-ve" "-ve" "+ve" "+ve" "+ve" "???"
# can also use vectors for true and false
if_else(x > 0, -x, x)
[1] -3 -2 -1  0 -1 -2 -3 NA
# Can also implement simple coalesce() version
x1 <- c(NA, 1, 2, NA)
y1 <- c(3, NA, 4, 6)
if_else(is.na(x1), y1, x1) # replacing NA values
[1] 3 1 2 6
# Can also nest if_else() in if_else()
if_else(x == 0, "0", if_else(x < 0, "-ve", "+ve"), "?")
[1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" "?"
Tip

With more conditions, switch to dyplr::case_when().

case_when()

Different computations for different conditions. Syntax: condition ~ output. condition must be a logical vector - when TRUE, output will be used:

case_when(
  x == 0 ~ "0",
  x > 0 ~ "+ve",
  x < 0 ~ "-ve",
  is.na(x) ~ "???"
)
[1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" "???"
# Can also specify default for all other values:
case_when(
  x < 0 ~ "-ve",
  x > 0 ~ "+ve",
  .default = "???"
)
[1] "-ve" "-ve" "-ve" "???" "+ve" "+ve" "+ve" "???"
# Note: if multiple conditions match, only first will be used:
case_when(
  x > 0 ~ "+ve",
  x > 2 ~ "big"
)
[1] NA    NA    NA    NA    "+ve" "+ve" "+ve" NA

Can also use variables on both sides of ~; can mix and match variables as needed.

flights |>
  mutate(
    status = case_when(
      is.na(arr_delay) ~ "cancelled",
      arr_delay < -30 ~ "very early",
      arr_delay < -15 ~ "early",
      abs(arr_delay) <= 15 ~ "on time",
      arr_delay < 60 ~ "late",
      arr_delay < Inf ~ "very late"
    ),
    .keep = "used"
  )

Compatible types

if_else() and case_when() require compatible types in output:

  • Numerical and logical vectors are compatible
  • strings and factors are compatible
  • Dates and date-times are compatible
  • NA is compatible with everything
