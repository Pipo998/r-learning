Making numbers

Most cases, numbers will be recorded in R’s numeric type: integer or double. Sometimes as strings. readr provides useful functions parsing strings into numbers: parse_double() and parse_number().

Use parse_double() when numbers have been written as strings:

x <- c("1.2", "5.6", "1e3")
class(x)
[1] "character"
parse_double(x)
[1]    1.2    5.6 1000.0
class(parse_double(x))
[1] "numeric"

Use parse_number() when strings contain non-numeric text that we want to ignore. Useful for currency data and percentages:

x <- c("$1,234", "USD 3,513", "59%")
class(x)
[1] "character"
parse_number(x)
[1] 1234 3513   59
class(parse_number(x))
[1] "numeric"

Arithmetic and recycling rules

Basics of arithmetic (x, -, *, /, ^). Recycling rules determine what happens with different size sides.

\(\rightarrow\) R handles mismatched lengths by recycling (repeating) the short vector:

x <- c(1, 2, 10, 20)
x / 5
[1] 0.2 0.4 2.0 4.0
# is shorthand for:
x / c(5, 5, 5, 5)
[1] 0.2 0.4 2.0 4.0

Want to recycle single numbers - BUT R will recylce any shorter length vector:

x * c(1, 2)
[1]  1  4 10 40
x * c(1, 2, 3)
Warning in x * c(1, 2, 3): longer object length is not a multiple of shorter
object length
[1]  1  4 30 20

Recycling rules apply to logical comparisons (==, <, <=, >, >=, !=):

Minimum and Maximum

Arithmetic functions work onm pairs, e.g., pmin() and pmax() - when given two or more variables, returns smallest or largest value in each row:

df <- tribble(
  ~x, ~y,
  1, 3, 
  5, 2, 
  7, NA,
)

df |>
  mutate(
    min = pmin(x, y, na.rm = TRUE),
    max = pmax(x, y, na.rm = TRUE)
  )
# A tibble: 3 × 4
      x     y   min   max
  <dbl> <dbl> <dbl> <dbl>
1     1     3     1     3
2     5     2     2     5
3     7    NA     7     7
# Note: summary functions min() and max() return single value from multiple observations

df |> 
  mutate(
    min = min(x, y, na.rm = TRUE),
    max = max(x, y, na.rm = TRUE)
  )
# A tibble: 3 × 4
      x     y   min   max
  <dbl> <dbl> <dbl> <dbl>
1     1     3     1     7
2     5     2     1     7
3     7    NA     1     7

Modular arithmetic

I.e. division that yields a whole number and a remainder.

%/% = integer division %% = computes remainder

Logarithms

Used when dealing with data ranging across multiple orders and converting exponetial growth to linear growth.

Three choices: log(), log2()(base 2), log10()(base 10).

Inverse of log() is exp().

Rounding

round(x)round number to nearest integer.

round(123.456)
[1] 123

Can control precision of rounding with second argument, digits, i.e. round(x, digits).

round(123.456, digits = 2)
[1] 123.46
round(123.456, digits = 1)
[1] 123.5
round(123.456, digits = -1) # rounds to nearest ten
[1] 120
round(123.456, digits = -2) # rounds to nearest hundred
[1] 100
round(123.456, digit = -3) # rounds to nearest thousand
[1] 0
# Banker's rounding, i.e. "round half to even":
round(c(1.5, 2.5))
[1] 2 2
# floor() and ceiling()
x <- 123.456

floor(x)
[1] 123
ceiling(x)
[1] 124
# Rounding down to nearest two digits
floor(x / 0.01) * 0.01
[1] 123.45
# Rounding up to nearest two digits
ceiling(x / 0.01) * 0.01
[1] 123.46
# Round to nearest multiple of 4
round(x / 4) * 4
[1] 124
round(x / 0.25) * 0.25
[1] 123.5

Cutting numbers into ranges

Use cut() to break up (aka bin) numeric vector into discrete buckets:

x <- c(1, 2, 5, 10, 15, 20)

cut(x, breaks = c(0, 5, 10, 15, 20))
[1] (0,5]   (0,5]   (0,5]   (5,10]  (10,15] (15,20]
Levels: (0,5] (5,10] (10,15] (15,20]
# breaks don't need evenly spacing
cut(x, breaks = c(0, 5, 10, 100))
[1] (0,5]    (0,5]    (0,5]    (5,10]   (10,100] (10,100]
Levels: (0,5] (5,10] (10,100]
# can supply labels
cut(x, 
  breaks = c(0, 5, 10, 15, 20), 
  labels = c("sm", "md", "lg", "xl")
)
[1] sm sm sm md lg xl
Levels: sm md lg xl
# Values outside the range:
y <- c(NA, -10, 5, 10, 30)
cut(y, breaks = c(0, 5, 10, 15, 20))
[1] <NA>   <NA>   (0,5]  (5,10] <NA>  
Levels: (0,5] (5,10] (10,15] (15,20]

General transformations

Ranks

See documentation for: dplyr::min_rank(), dplyr::row_number(), dplyr::dense_rank()

x <- c(1, 5, 5, 17, 22, NA)

min_rank(x) # gives tie same smallest value
[1]  1  2  2  4  5 NA
dense_rank(x) # works like min_rank() but leaves no gap
[1]  1  2  2  3  4 NA
row_number(x) # gives every input unique rank
[1]  1  2  3  4  5 NA
# equivalent to
rank(x, ties.method = "first", na.last = "keep") # see documentation
[1]  1  2  3  4  5 NA
min_rank(desc(x)) # largest values get smallest rank
[1]  5  3  3  2  1 NA
# Example with df
df <- tibble(x = x)

df |>
  mutate(
    row_number = row_number(x),
    dense_rank = dense_rank(x),
    percent_rank = percent_rank(x),

Offsets

dplyr::lag() and dplyr::lead() allow to refer to values before or after “current” value - returns vector of same length:

x <- c(2, 5, 11, 11, 19, 35)

lag(x)
[1] NA  2  5 11 11 19
lead(x)
[1]  5 11 11 19 35 NA
# Difference between current and previous value
x - lag(x)
[1] NA  3  6  0  8 16
# When does current value change?
x == lag(x)
[1]    NA FALSE FALSE  TRUE FALSE FALSE
Note

Call lead or lag by more than one position by using second argument, n.

Numeric summaries

Center

Alternativ to mean() is function median() - finding values in middle of vector.

For symmetric distribution, rather report mean, for skewed distributions usually report median.

flights |>
  group_by(year, month, day) |>
  summarize(
    mean = mean(dep_delay, na.rm = TRUE),
    median = median(dep_delay, na.rm = TRUE),
    n = n(),
    .groups = "drop"
  ) |>
  ggplot(
    mapping = aes(x = mean, y = median)
  ) +
  geom_abline(slope = 1, intercept = 0, color = "white", linewidth = 2) +
  geom_point()

Distributions

Visualize distribution before comitting to summary statistics - avoid missing important insights by choosing wrong summary

E.g., plot shows 365 frequency polygons of dep_delay, one for each day, are overlaid. Distribution seems to follow common pattern - it is fine to use same summary for each day:

flights |>
  filter(dep_delay < 120) |> 
  ggplot(aes(x = dep_delay, group = interaction(day, month))) + 
  geom_freqpoly(binwidth = 5, alpha = 1/5)

With mutate()

Summary functions typically paired with summarize(). Because of recycling, can be paired with mutate().

  • x / sum(x): computes proportion of total
  • (x - mean(x)) / sd(x): computes Z-score
  • (x - min(x)) / (max(x) - min(x)): standardize to range [0, 1]
  • x / first(x): compute index based on first observation.
